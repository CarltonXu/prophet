## 1. 产品介绍

Prophet 是一个自动化资源采集、分析和管理的工具集，专为云迁移与云灾备前期技术调研设计。项目提供现代化的 Web 界面和强大的命令行工具，支持对物理机、VMware 环境的全面采集和分析，帮助用户快速了解源端基础设施状况，确保迁移和灾备方案的可行性。

### 1.1 核心价值

- **自动化采集**：通过多种协议（nmap、Ansible、WMI、VMware API）自动采集主机详细信息
- **可视化分析**：提供直观的 Web 界面，支持应用拓扑可视化和管理
- **数据脱敏**：自动移除敏感信息，确保数据安全
- **容器化部署**：一键部署，开箱即用，减少环境依赖

### 1.2 应用场景

- 云迁移前期调研：全面了解源端基础设施，制定迁移方案
- 云灾备规划：评估灾备可行性，预测数据传输时间
- 基础设施管理：统一管理主机、应用和依赖关系
- 技术文档生成：自动生成技术调研报告

## 2. 业务流程

### 2.1 整体业务流程

Prophet 系统的主要业务流程如下：

1. **资源发现阶段**

   - 通过网络扫描发现网络中的主机设备
   - 通过虚拟化平台同步获取虚拟机信息
   - 手动添加主机设备

2. **数据采集阶段**

   - 配置主机认证信息（SSH、WMI 等）
   - 创建采集任务，批量采集主机详细信息
   - 查看采集进度和结果

3. **数据管理阶段**

   - 查看和管理主机信息
   - 为主机添加标签进行分类
   - 创建应用并梳理业务拓扑关系

4. **数据分析阶段**
   - 通过仪表盘查看系统统计信息
   - 导出数据进行分析和报告生成

### 2.2 典型使用流程

**场景一：网络扫描发现主机**

1. 登录系统
2. 进入"网络扫描"页面
3. 创建扫描任务，指定扫描目标
4. 等待扫描完成
5. 查看扫描结果，将发现的主机添加到设备列表

**场景二：从虚拟化平台同步主机**

1. 登录系统
2. 进入"虚拟化平台"页面
3. 添加 VMware 平台配置
4. 执行平台资源同步
5. 同步的主机自动添加到设备列表

**场景三：采集主机详细信息**

1. 进入"设备列表"页面
2. 为主机配置认证信息（SSH 用户名/密码或密钥）
3. 选择主机，执行采集操作
4. 进入"采集任务"页面查看采集进度
5. 采集完成后，在设备列表中查看详细信息

**场景四：应用拓扑管理**

1. 进入"应用管理"页面
2. 创建新应用
3. 进入应用详情页面
4. 在可视化画布中添加主机和关系
5. 调整布局，保存拓扑图

## 3. 产品功能说明

### 3.1 登录与注册

#### 3.1.1 登录页面

![登录页面](/docs/images/manual/01-login-page-zh.png)

**功能说明：**

- 用户可以通过用户名和密码登录系统
- 系统提供验证码功能，防止暴力破解
- 支持自动识别验证码（需要用户协助）
- 首次使用需要先注册账号

**操作步骤：**

1. 访问系统登录页面（默认地址：http://localhost:3000）
2. 输入用户名和密码
3. 输入验证码（系统会自动识别）
4. 点击"登录"按钮

#### 3.1.2 注册页面

![注册页面](/docs/images/manual/02-register-page-zh.png)

**功能说明：**

- 新用户可以通过注册页面创建账号
- 需要填写用户名、邮箱和密码
- 同样需要输入验证码

**操作步骤：**

1. 在登录页面点击"注册"链接
2. 填写用户名、邮箱和密码
3. 输入验证码
4. 点击"注册"按钮

### 3.2 仪表盘

![仪表盘页面](/docs/images/manual/03-dashboard-zh.png)

**功能说明：**
仪表盘页面提供系统概览和统计数据，包括：

- 总设备数：显示系统中所有主机的总数，区分物理机和虚拟机
- 已采集设备：显示已完成数据采集的主机数量
- 虚拟化平台：显示已配置的虚拟化平台数量
- 应用数量：显示已创建的应用数量
- 最近扫描任务：显示最近执行的扫描任务列表及其状态

**主要功能：**

- 刷新按钮：手动刷新统计数据
- 任务状态显示：显示任务的执行状态（等待中、运行中、已完成等）
- 任务进度显示：显示任务的执行进度百分比

### 3.3 设备列表

设备列表是系统的核心功能模块，提供了丰富的主机管理功能。

#### 3.3.1 列表视图

![设备列表-列表视图](/docs/images/manual/04-hosts-list-view-zh.png)

**功能说明：**

- 以表格形式展示所有主机设备
- 支持多列信息展示（IP、主机名、OS 类型、发行版、OS 版本、设备类型、CPU 核心、内存、磁盘、采集状态、来源、标签等）
- 每行显示一个主机的基本信息

#### 3.3.2 树形视图

![设备列表-树形视图](/docs/images/manual/05-hosts-tree-view-zh.png)

**功能说明：**

- 以树形结构展示主机设备
- 可以按照平台、标签等维度进行分组展示
- 支持展开和折叠节点
- 便于查看主机之间的层级关系

#### 3.3.3 视图切换

**功能说明：**

- 提供列表视图和树形视图两种展示方式
- 通过视图切换按钮可以快速切换展示模式

#### 3.3.4 搜索功能

![设备列表-搜索功能](/docs/images/manual/07-hosts-search-zh.png)

**功能说明：**

- 支持多字段搜索：可以按 IP、主机名、MAC 地址、厂商、标签等字段进行搜索
- 字段选择器：点击搜索框可以选择搜索字段
- 标签搜索：支持通过下拉菜单选择标签进行筛选
- 实时搜索：输入关键词后自动进行搜索

**操作步骤：**

1. 点击搜索框
2. 选择搜索字段（可选，默认为全部字段）
3. 输入搜索关键词
4. 系统自动显示搜索结果

#### 3.3.5 筛选功能

![设备列表-筛选功能](/docs/images/manual/08-hosts-filters-zh.png)

**功能说明：**

- 提供多个筛选条件：
  - OS 类型：Linux、Windows、VMware ESXi 等
  - 设备类型：主机、虚拟机、网络设备
  - 采集状态：未采集、采集中、已完成、失败
  - 来源：扫描、平台、手动
  - 设备性质：物理机、虚拟机
  - 标签：按标签筛选
- 筛选条件可以组合使用
- 显示已选筛选条件，可以快速清除

#### 3.3.6 添加设备

![设备列表-添加设备弹窗](/docs/images/manual/09-hosts-add-modal-zh.png)

**功能说明：**

- 支持手动添加主机设备
- 需要填写的基本信息：
  - IP 地址（必填）
  - 主机名（可选）
  - MAC 地址（可选）
  - 厂商（可选）
  - OS 类型（可选）
  - OS 版本（可选）
  - 设备类型（可选）
  - 是否物理机（可选）
  - 虚拟化平台信息（可选）
  - 认证信息（可选）

**操作步骤：**

1. 点击"添加设备"按钮
2. 填写设备信息
3. 点击"保存"按钮

#### 3.3.7 自定义列

![设备列表-自定义列弹窗](/docs/images/manual/10-hosts-custom-columns-zh.png)

**功能说明：**

- 允许用户自定义表格显示的列
- 提供列模板：基础视图、详细视图、硬件视图
- 支持按类别选择列：基础信息、硬件信息、状态信息、元数据
- 可以重置为默认列配置

**操作步骤：**

1. 点击自定义列按钮
2. 选择列模板或手动选择列
3. 点击"保存"按钮

#### 3.3.8 批量操作

![设备列表-批量操作菜单](/docs/images/manual/11-hosts-batch-actions-zh.png)

**功能说明：**
当选中多个主机后，可以执行以下批量操作：

- 更新认证：批量更新选中主机的认证信息
- 采集：批量创建采集任务
- 批量加入应用：将选中的主机添加到指定应用
- 管理标签：批量添加或移除标签
- 导出 Excel：导出选中主机的数据
- 删除设备：批量删除选中的主机

**操作步骤：**

1. 勾选需要操作的主机（或点击表头的全选复选框）
2. 点击"批量操作"按钮
3. 选择要执行的操作
4. 按照提示完成操作

#### 3.3.9 导入数据

![设备列表-导入数据弹窗](/docs/images/manual/12-hosts-import-modal-zh.png)

**功能说明：**

- 支持通过 CSV 文件批量导入主机数据
- 提供三种模板类型：
  - 空模板：仅包含字段结构，适合手工填写
  - 未采集设备模板：导出所有未采集设备，便于批量补录
  - 当前筛选结果：根据当前筛选条件导出设备清单
- 导入过程显示进度和结果统计
- 导入错误会显示详细的错误信息

**操作步骤：**

1. 点击"导入 CSV"按钮
2. 选择模板类型并下载模板
3. 按照模板格式填写数据
4. 上传填写好的 CSV 文件
5. 点击"开始导入"按钮
6. 查看导入结果

#### 3.3.10 导出数据

![设备列表-导出数据弹窗](/docs/images/manual/13-hosts-export-modal-zh.png)

**功能说明：**

- 支持导出选中设备或所有设备（当前筛选）
- 提供多种导出模板：
  - 概要模板：包含关键基础信息
  - 详细模板：包含硬件、存储和网络详细信息
  - 全部字段：导出所有支持的字段
- 支持自定义选择导出字段
- 导出格式为 Excel

**操作步骤：**

1. 选择要导出的设备（可选，不选择则导出所有）
2. 点击"导出 Excel"按钮
3. 选择导出模板或自定义字段
4. 点击"导出"按钮

#### 3.3.11 主机详情

![设备列表-主机详情页面](/docs/images/manual/14-hosts-details-zh.png)

主机详情页面显示主机的完整信息，包括：

## 4. 达成的目标

Prophet 系统通过自动化的资源采集和管理，帮助用户达成以下目标：

### 4.1 自动化资源发现

- 通过网络扫描自动发现网络中的主机设备
- 通过虚拟化平台 API 自动同步虚拟机信息
- 减少人工录入的工作量，提高效率

### 4.2 全面的数据采集

- 支持多种采集方式：nmap 扫描、Ansible、WMI、VMware API
- 采集主机详细信息：硬件配置、操作系统、网络、存储等
- 支持批量采集，提高采集效率

### 4.3 可视化管理

- 提供直观的 Web 界面进行数据管理
- 支持应用拓扑可视化，清晰展示业务架构
- 支持多种视图模式，满足不同场景需求

### 4.4 数据安全

- 自动数据脱敏处理
- 支持数据加密存储
- 提供数据导出功能，便于离线分析

### 4.5 云迁移/灾备支持

- 为云迁移和灾备提供完整的基础设施信息
- 支持生成技术调研报告
- 帮助评估迁移和灾备的可行性

## 5. 常见问题处理

### 5.1 登录问题

**问题：无法登录系统**

**解决方案：**

1. 检查用户名和密码是否正确
2. 检查验证码是否输入正确
3. 如果忘记密码，需要联系管理员重置
4. 检查浏览器是否启用了 Cookie

### 5.2 采集失败问题

**问题：主机采集失败**

**解决方案：**

1. 检查主机的认证信息是否正确配置
2. 检查网络连通性（SSH、WMI 等端口是否开放）
3. 检查主机的防火墙设置
4. 查看采集任务的错误信息，根据错误提示进行排查

### 5.3 扫描任务问题

**问题：扫描任务执行失败或没有结果**

**解决方案：**

1. 检查扫描目标格式是否正确
2. 检查网络连通性
3. 检查系统是否安装了 nmap 工具
4. 查看任务详情中的错误信息

### 5.4 平台同步问题

**问题：虚拟化平台同步失败**

**解决方案：**

1. 检查平台配置信息是否正确（主机地址、端口、用户名、密码）
2. 使用"测试连接"功能验证配置
3. 检查网络连通性
4. 检查平台 API 权限

### 5.5 数据导入问题

**问题：CSV 导入失败**

**解决方案：**

1. 检查 CSV 文件格式是否正确
2. 确保必填字段（IP）已填写
3. 检查数据格式是否符合要求
4. 查看导入结果中的错误信息，根据提示修正数据

### 5.6 性能问题

**问题：页面加载缓慢**

**解决方案：**

1. 检查网络连接
2. 减少单次显示的数据量（使用分页）
3. 使用筛选功能减少显示的数据
4. 检查系统资源使用情况

### 5.7 浏览器兼容性

**问题：某些功能在浏览器中无法正常使用**

**解决方案：**

1. 推荐使用 Chrome、Firefox、Edge 等现代浏览器
2. 确保浏览器版本较新
3. 清除浏览器缓存后重试
4. 检查浏览器是否启用了 JavaScript

### B. 数据格式说明

#### CSV 导入格式

必填字段：

- `ip`：IP 地址

可选字段：

- `hostname`：主机名
- `mac`：MAC 地址
- `vendor`：厂商
- `os_type`：操作系统类型
- `os_version`：操作系统版本
- `device_type`：设备类型
- `is_physical`：是否物理机（是/否 或 true/false）
- `vt_platform`：虚拟化平台
- `username`：账号
- `password`：密码
- `ssh_port`：SSH 端口
- `key_path`：密钥路径
- `tags`：标签（多个标签用逗号分隔）
